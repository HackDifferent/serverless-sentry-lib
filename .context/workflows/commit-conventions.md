# Commit Conventions

## Format

This repository uses **plain descriptive commits** with no strict ticket-ID or conventional-commit prefix requirement. The author mixes styles freely.

## Style Guidelines for This Repo

1. **No ticket ID required** — there is no Jira/Linear project key in use
2. **Sentence case, imperative or past tense** both appear — either is acceptable
3. **Version bump commits** are bare version numbers (`2.5.2`) generated by `npm version`
4. **Occasional `type:` prefix** (`docs:`) but not enforced
5. **No body or footer** conventions enforced — keep it concise

## Well-Formed Examples

```
Fixed invalid `@sentry/integrations` peer dependency definition.
`@sentry/integrations` is a peer dependency now.
Rewrite library using Typescript; updated to latest dependencies
Renamed SentryLambdaWrapper.handle to `withSentry`, simplified API
Prepared next release
```

## What to Avoid

- Do not add `feat:`, `fix:`, `chore:` prefixes unless the change is clearly documentation (`docs:`)
- Do not include issue numbers (not tracked in git messages here)
